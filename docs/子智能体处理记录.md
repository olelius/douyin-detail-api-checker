## 2026-06-24 非致命候选响应解析失败回归补强记录

### 回归背景

- 主线程在真实小批量验证中再次观察到前 3 条被判为 `待确认 / error`。
- 输出 Excel 备注中的错误为候选 detail API 响应解析失败：
  - `detail API 响应 JSON 解析失败：response.json: Target page, context or browser has been closed`
  - `detail API 响应 JSON 解析失败：response.json: Protocol error (Network.getResponseBody): No resource with given identifier found`

### 本次确认的根因

- 候选响应解析失败本质上属于非致命事件，尤其发生在页面关闭后或响应体已不可再次提取时。
- 分类器虽然已有“旧式非致命解析失败不阻断 DOM 兜底”的兼容逻辑，但 detector 侧缺少单独的行为约束测试，后续改动仍可能把这类错误重新升级为 fatal error。
- `apiParseErrors` 既可能是字符串数组，也可能是对象数组；分类器需要同时兼容这两种记录形态。

### 本次修复

- `src/detector.js`
  - 新增 `recordApiParseError(result, responseUrl, error)`。
  - 候选 `response.json()` 失败统一只写入 `apiParseErrors` 和 `debugMessages`，不写入 `result.error`。
  - `debugMessages` 追加 `[detail-api][non-fatal]` 调试信息，便于排查但不影响分类结果。
- `src/classifier.js`
  - `isNonFatalApiParseError()` 改为同时兼容字符串数组和对象数组形式的 `apiParseErrors`。
  - 仅当 `error` 明确是 `detail API 响应 JSON 解析失败` 且存在解析失败记录时，才将其视为非致命并继续走 DOM 兜底。
- `test/detector.test.js`
  - 新增覆盖：候选响应 JSON 解析失败只记录到非致命字段，不写入 fatal `error`。
- `test/classifier.test.js`
  - 新增覆盖：`apiParseErrors` 为对象数组时，DOM 失效文本仍应返回 `失效 / dom_text`。

| `cmd /c npm test -- test/detector.test.js test/classifier.test.js`（补强后） | 通过：14 个子测试全部通过 |

### 结论

- 候选 JSON 解析失败现已被显式限定为非致命事件。
- 即使未来 `apiParseErrors` 记录形态变化，只要 DOM 兜底文本明确，分类器也不会再优先输出 `待确认 / error`。

## 2026-06-24 上传 Excel 500 bug 修复记录

### 任务边界

- 未修改旧 Python 目录 `methods/python_playwright_dom_text`。

### 根因

- `readInputWorkbook()` 返回的 `inputData.rows` 每一行包含 `originalRow`。
- `originalRow` 是 ExcelJS Row 对象，内部持有 worksheet/workbook 等循环引用。
- `src/web-server.js` 的 multipart 上传路径把 `rows: inputData.rows` 直接传给任务队列。
- `src/task-manager.js` 在 `buildTask()` 中会通过 `JSON.stringify()` 克隆 rows，因此遇到 ExcelJS 循环对象后抛出 `TypeError: Converting circular structure to JSON`，最终接口返回 500。

### 修复内容

- 在 `src/web-server.js` 增加 `toDetectRows()`，将上传 Excel 解析出的 rows 转为轻量检测行，只保留 `rowNumber` 和 `url`。
- `readMultipartTaskPayload()` 继续保留完整 `inputData`，供后续导出沿用原 workbook。
- `readMultipartTaskPayload()` 传给 task manager 的 `rows` 改为轻量 rows，不再携带 `originalRow`、ExcelJS Row、worksheet/workbook 等循环对象。
- 在 `test/web-server.test.js` 的 fake task manager 中增加断言：`payload.rows` 必须可 `JSON.stringify()`，且首行不能包含 `originalRow`。
- 先运行上传测试观察到修复前稳定返回 500，再修复并确认测试转绿。
